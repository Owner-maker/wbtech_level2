Что выведет программа? Объяснить вывод программы. Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
	"fmt"
	"os"
)

func Foo() error {
	var err *os.PathError = nil
	return err
}

func main() {
	err := Foo()
	fmt.Println(err)
	fmt.Println(err == nil)
}
```

Ответ:
```
Вывод:
<nil>
false

Возвращаемое значение при вызовые функции foo() будет обернуто в интерфейс типа error,
где в поле tab будет находиться указатель на тип ошибки (os.PathError), а data будет nil

При выводе переменной err мы получаем nil, так как data = nil (то есть у ошибки нет содержания),
но при сравнении err с nil, мы получаем false, так как в Golang для того, чтобы значение интерфейса
равнялось nil оба поля (tab и data) должны быть nil, а в нашем случае tab != nil.

Пустой интерфейс отличается от обычного тем , что у первого отсутствует поле tab (с метаинформацией о типе и 
списком методов для реализации), а только лишь сам тип + data (ссылка на реальную область памяти, где лежат 
данные изначального объекта.

Именно поэтому в interface{} можно "завернуть" что угодно, т.к. любой тип имплементирует автоматически пустой
интерфейс без методов.
```
