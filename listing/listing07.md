Что выведет программа? Объяснить вывод программы.

```go
package main

import (
	"fmt"
	"math/rand"
	"time"
)

func asChan(vs ...int) <-chan int {
	c := make(chan int)

	go func() {
		for _, v := range vs {
			c <- v
			time.Sleep(time.Duration(rand.Intn(1000)) * time.Millisecond)
		}

		close(c)
	}()
	return c
}

func merge(a, b <-chan int) <-chan int {
	c := make(chan int)
	go func() {
		for {
			select {
			case v := <-a:
				c <- v
			case v := <-b:
				c <- v
			}
		}
	}()
	return c
}

func main() {

	a := asChan(1, 3, 5, 7)
	b := asChan(2, 4 ,6, 8)
	c := merge(a, b )
	for v := range c {
		fmt.Println(v)
	}
}
```

Ответ:
```
Будут выведены все числа, переданные в качестве аргументов при вызове функции asChan (вызванной дважды), но
с совершенно разным порядком, причиной этого является искусственная задержка функцией time.Sleep с переданным
аргументом в виде случайного значения в пределах 1 секунды. Остальные значения будут выводится нулями до бесконечности.

Сначала дважды вызывается функция asChan, порождающая две пишущие горутины в два разных канала переданные значения,
а вызванная функция merge порождает еще одну горутину, которая с использованием бесконечного цикла и оператора select по
очереди читает значения с двух каналов (в зависимости от того, в какой канал пришло значение, именно здесь задержка 
и играет свою роль, благодаря чему читаются значения в случайном порядке) и пишет полученные значения в новый канал.

В главном потоке main циклом читаются значения с итогового канала и значения выводятся в консоль.
В силу того, что оба канала, с которых производится чтение закрываются после того, как все числа были переданы,
а горутина, которая занимается чтением с этих канало и записью в другой канал не завершается (бесконечный цикл), то
из закрытых канало вычитываются значения по умолчанию - нули, до бесконечности.
```
